# `data/` — Runtime Data Directory

This folder holds all inputs and outputs produced by the pipeline. See the [main workflow](../README.md#workflow) for the full pipeline steps.

---

## Content

Only `oam_catalog.gpkg` and `AOI.gpkg` are committed — everything else is gitignored and regenerated by the pipeline.
Some files are gitignored and can be regenerated when running the pipeline locally.

```
data/
|
+-- oam_catalog.gpkg           <- OAM query result — scene footprints + metadata
+-- oam_catalog.csv            <- same data in flat CSV
+-- AOI.gpkg                   <- primary output used for metrics and downstream analysis
|
+-- imagery/                   <- downloaded GeoTIFFs
|   +-- <oam_id>.tif
|
+-- tiles/                     <- tile grids, one GPKG per scene
|   +-- <oam_id>_tiles.gpkg
|
+-- dataset/                   <- YOLO dataset for training
|   +-- train/  waste/  background/
|   +-- val/    waste/  background/
|   +-- test/   waste/  background/
|
+-- auxiliary/                 <- third-party data, downloaded at runtime
|   +-- africa_geodata.parquet <- MillionNeighborhoods Africa (~8 GB)
|   +-- shdi_national.csv      <- Global Data Lab SHDI
|
+-- predictions/
|   +-- green/                 <- SAM greenery polygons
|       +-- <oam_id>_green.gpkg
|
+-- models/
|   +-- train/                 <- YOLO training run outputs
|       +-- weights/
|           +-- best.pt       <- best model checkpoint (from 02_predict.py --model)
|           +-- last.pt
|
+-- results/                   <- evaluation outputs and figures

```

## Files and Folders

### `oam_catalog.gpkg` · `oam_catalog.csv`

**Produced by:** `01_data_acquisition_preprocessing/01_query_oam_catalog.py`

Scene-level metadata for all OpenAerialMap entries that pass the initial quality filters (GSD 3.5–6.0 cm, area ≥ 1 km²).

| Column | Description |
|---|---|
| `oam_id` | Unique OAM scene identifier |
| `title` | Scene title from OAM API |
| `gsd` | Ground sampling distance (cm/px) |
| `area_km2` | Scene footprint area (km²) |
| `acquisition_start` | Acquisition date |
| `provider` | Data provider name |
| `country` / `city` | Populated by `--geocode` flag (Nominatim) or filled manually |
| `geometry` | Scene footprint polygon (EPSG:4326) |

> After generating this file, **manually** inspect scene footprints in QGIS, remove low-quality scenes, and save the final AOI selection as `data/oam_AOI.gpkg`. This study retained 29 AOIs.

---

### `imagery/<oam_id>.tif`

**Produced by:** `01_data_acquisition_preprocessing/02_download_and_tile.py`

Cloud-optimised GeoTIFFs downloaded from OAM. Where a single AOI has multiple overlapping scenes, those are merged into one GeoTIFF (most-recent acquisition on top). Files are large (typically 1–8 GB each) and are therefore gitignored.

---

### `tiles/<oam_id>_tiles.gpkg`

**Produced by:** `01_data_acquisition_preprocessing/02_download_and_tile.py`  
**Extended by:** `02_model_training/02_predict.py`

One GeoPackage per scene. Each row represents one 5 × 5 m tile. Some columns are added in subsequent pipeline steps.

| Column | Type | Added by | Description |
|---|---|---|---|
| `tile_id` | int | tiling step | Unique tile identifier within the scene |
| `oam_id` | str | tiling step | Parent scene OAM ID |
| `row` / `col` | int | tiling step | Grid position |
| `filename` | str | tiling step | PNG path used during dataset creation |
| `label` | int / null | **manual** | Ground truth: `1` = waste, `0` = background, null = unlabeled |
| `pred_class` | str | predict step | YOLO output: `"waste"` or `"background"` |
| `confidence` | float | predict step | YOLO softmax score for the predicted class |
| `sam_greenery_pct` | float | predict step | % of tile area covered by SAM greenery polygons |
| `pred_class_green` | str | predict step | `"green"` if `sam_greenery_pct ≥ 25 %`, else `"background"` |

> **Manual step:** open each `_tiles.gpkg` in QGIS and set the `label` column for ~100 waste and ~100 background tiles per region before running `03_create_yolo_dataset.py`.

---

### `dataset/{train,val,test}/{waste,background}/*.png`

**Produced by:** `01_data_acquisition_preprocessing/03_create_yolo_dataset.py`

128 × 128 px PNG crops in a YOLO classification directory layout. Approximately 5 800 tiles in total across all 29 regions, split 70 / 15 / 15 % per region (seed = 0). Gitignored because the crops can always be regenerated from the source GeoTIFFs and labeled GPKGs.

---

### `auxiliary/africa_geodata.parquet`

**Produced by:** `01_data_acquisition_preprocessing/04_download_auxiliary_data.py`

MillionNeighborhoods Africa urban-morphology dataset (~8 GB). Contains one row per urban block across the African continent.

| Column | Description |
|---|---|
| `k_complexity` | Street-network complexity index |
| `worldpop_population_un_density_hectare` | UN-adjusted population density (persons/ha) |
| `geometry` | Block polygon |


### `auxiliary/shdi_national.csv`

**Produced by:** `01_data_acquisition_preprocessing/04_download_auxiliary_data.py`

Sub-national Human Development Index from the [Global Data Lab](https://globaldatalab.org/), sourced from Zenodo record [17467221](https://zenodo.org/record/17467221). Used at country level (latest year per country).

---

### `predictions/green/<oam_id>_green.gpkg`

**Produced by:** `02_model_training/02_predict.py --sam greenery`

One GeoPackage per scene containing polygon geometries of areas classified as vegetation by Meta SAM 3. The polygons are intersected with the tile grid in `02_predict.py` to populate `sam_greenery_pct` and `pred_class_green` in the corresponding `_tiles.gpkg`. Gitignored (regenerated by running `--sam greenery`).

---

### `AOI.gpkg`

**Produced by:** `03_analysis/01_calculate_aoi_metrics.py`

**Committed to the repository.** One row per study AOI (29 total). This is the primary output used for figure generation and statistical analysis. Some columns are added in subsequent pipeline steps.

| Column | Description |
|---|---|
| `oam_id` | Scene identifier |
| `waste_pct` | RSWCI (%) — fraction of tiles predicted as waste |
| `green_pct` | Vegetation coverage (%) — fraction of tiles predicted as green |
| `k_complexity_weighted` | Coverage-weighted street-network complexity |
| `worldpop_population_un_density_hectare_weighted` | Coverage-weighted population density (persons/ha) |
| `shdi` | National Sub-national HDI (latest available year) |
| `country` / `city` | Country and city name (from OAM catalog) |
| `State/Region` / `Region Name` | Administrative subdivision labels |
| `geometry` | AOI footprint polygon |
